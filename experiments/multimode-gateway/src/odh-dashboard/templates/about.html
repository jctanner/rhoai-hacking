About ODH Gateway

What is ODH Gateway?

The Open Data Hub Gateway is a centralized proxy system that provides unified access to all your data science and machine learning tools through a single entry point.

This dashboard serves as the main landing page and service directory, automatically discovering and listing all available services in your ODH environment.

Key Features

  • Centralized Authentication
    Single sign-on for all services
  • Automatic Service Discovery
    Services are discovered automatically
  • Dynamic Routing
    Smart path-based routing to services
  • OIDC & OpenShift Auth
    Support for multiple auth providers
  • Hot Reload Configuration
    Updates without service restarts
  • Kubernetes Native
    Built for cloud-native environments

System Architecture

The ODH Gateway system consists of three main components:

ODH Gateway
Reverse proxy with authentication
Gateway Operator
Manages gateway deployments and service discovery
Notebook Operator
Creates and manages Jupyter notebooks

How Service Discovery Works

  1. Services are deployed with special annotations:
    odhgateway.opendatahub.io/enabled: "true"
  2. The Gateway Operator watches for these annotated services
  3. Routing configuration is automatically updated in the gateway
  4. Services become accessible through the centralized gateway
  5. This dashboard displays all discovered services
Example Annotations:
annotations:
  odhgateway.opendatahub.io/enabled: "true"
  odhgateway.opendatahub.io/route-path: "/notebooks"

Technical Details

Dashboard Technology:
  • Flask web framework
  • Jinja2 templating
  • Bootstrap 5 UI
  • RESTful API endpoints
Gateway Features:
  • Dynamic route configuration
  • Path-based routing with fallbacks
  • Authentication middleware
  • Health check endpoints
This is an experimental project to validate the centralized gateway approach for ODH workloads. Individual components may be moved to separate repositories if this approach is approved.
